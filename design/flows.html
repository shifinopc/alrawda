Every transaction and master-data screen follows the same layout as today: a List on the left and a Detail form on the right with New, Save and Delete actions. The flows below reflect that pattern.

2. Login & Main Navigation

Entry point
  1. User opens the application in a browser and logs in (username, password, optional 2FA).
  2. The system checks credentials and the user’s role.
  3. On success, the Dashboard opens with only the menus the role is allowed to use.
  4. From the top bar the user opens any menu — Transaction (Basic / Activity / Reports) or Settings — each opening as a tab.

3. Package

Transaction → Basic → Package

Maintains travel packages and their rates (e.g. “5 Days Umrah With Visa — 1050”).

  1. Open Package; the list of existing packages loads with search/filter.
  2. Click New for a new package, or select a row to edit.
  3. Enter Package Name (English and Arabic) and Rate.
  4. Click Save; the list refreshes. Use Delete to deactivate an unused package.

4. Visa Type

Transaction → Basic → Visa Type

Maintains the visa types used on invoices and receipts (e.g. Umrah Visa, Tourist Visa).

  1. Open Visa Type; existing types load in the list.
  2. Click New or select a row.
  3. Enter the Visa Type name and Save; the list refreshes.

5. Nationality

Transaction → Basic → Nationality

Maintains the countries / nationalities list (country name, short name, Arabic name).

  1. Open Nationality; the country list loads with search/filter.
  2. Click New or select a row.
  3. Enter Country Name, Short Name and Name in Arabic.
  4. Click Save; the list refreshes.

6. Invoice

Transaction → Activity → Invoice

Creates the sale: customer, package, passengers and amounts.

  1. Open Invoice; the invoice list loads. Click New.
  2. Fill the header: customer name, nationality, mobile 1/2, package, departure date, no. of passengers/seats/visas, room type.
  3. Add passenger rows (name, visa type, visa-required).
  4. The system computes Amount, Discount and Net Amount from the package rate.
  5. Click Save. If approval is required, the invoice goes to the approver as Pending; otherwise it can be printed straight away.
  6. Print the bilingual invoice via the dynamic print engine.

7. Receipt

Transaction → Activity → Receipt

Collects money against an invoice and prints the bilingual Receipt Voucher (سند قبض).

  1. Open Receipt and click New.
  2. Select the invoice; customer, package and passenger details auto-fill.
  3. The system shows invoice amount, previous balance and remaining balance.
  4. Enter the received amount and mode of payment (cash / bank).
  5. Click Save, then Print the bilingual Receipt Voucher; the balance updates.

8. Payment

Transaction → Activity → Payment

Records office expenses and customer refunds.

  1. Open Payment and click New; choose Type: Expense or Refund.
  2. Expense — enter paid-to, mobile, narration, amount (e.g. office, salary, taxi, printing).
  3. Refund — select the invoice and the amount to refund.
  4. Click Save and Print the voucher. A refund reduces the linked invoice’s balance.

9. All Report

Transaction → Reports → All Report

One report screen produces all six report types by selection and date range.

  1. Open All Report.
  2. Choose the report type with the radio button: Income Summary, Pending, Income Detail, Passengers List, Departure Date wise, or Expense.
  3. Set the Start Date and End Date.
  4. Click Print; the report renders to preview, then export to PDF or Excel.

10. Settings

Top bar → Settings

Administrator area for company details, users, and the dynamic field / print configuration.

  1. Open Settings (admin only).
  2. Company info & branding — name, address, contacts, logo used on all vouchers.
  3. Users & roles — create users and assign role-based permissions.
  4. Dynamic fields — add/edit form fields on any screen without code; map printable fields into templates.
  5. Print templates — edit invoice, receipt, payment and report layouts (bilingual); Save to publish.
